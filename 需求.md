❄️ Snow Anti-Melt Module（需求文档 · 简洁版）

目标：
阻止 雪层（SNOW） 和 雪块（SNOW_BLOCK） 的融化，但不影响 冰类方块 的正常融化行为。
火把、灯、太阳等导致的雪融化全部被禁止。

1. 功能描述

阻止 雪层 SNOW 和 雪块 SNOW_BLOCK 在光照或环境变化下变成 AIR。

不拦截其他方块的融化，包括：

ICE

PACKED_ICE

BLUE_ICE

FROSTED_ICE

其他任何会因光照变成水或空气的方块等

模块仅影响 “雪 → 空气” 这一种行为。

2. 事件逻辑要求

监听 BlockFadeEvent：

如果 event.block.type 是 SNOW 或 SNOW_BLOCK

且 event.newState.type 是 AIR

则 event.isCancelled = true

伪逻辑：

if blockType in [SNOW, SNOW_BLOCK] and newType == AIR:
cancel event
else:
allow normal behavior

3. 核心规则

不修改冰块行为。

不处理太阳光照、火把光照等来源区分，只要触发融化事件就拦截。

不影响玩家破坏雪方块（那属于 BlockBreakEvent，不会被拦截）。

对粉雪（POWDER_SNOW）不做处理（它本身不因光照融化）。

4. 代码结构建议
   Listener

SnowAntiMeltListener

注册为常规的 Bukkit/Folia 事件监听器。

配置（可选）
snow-anti-melt:
enabled: true


若 enabled = false 则不拦截事件。

5. 性能要求

事件处理必须是 O(1)，不能执行数据库/IO。

仅做简单类型判断和取消事件。

6. 输出行为

无日志、无消息，隐藏式功能。
游戏内行为表现：雪不会因火把或光照融化。
