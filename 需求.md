📌 项目简介

开发一个轻量级 Minecraft 模块，为 PlaceholderAPI 注册一个自定义占位符：
%tsl_blocks_placed_total%
该占位符返回玩家累计放置的方块总数，基于原版统计（Statistic.USE_ITEM）计算，不使用事件监听、不持久化数据、不做数据库操作。
此插件的主要用途是与 Topper 搭配，用于排行榜统计。
🎯 功能要求
1️⃣ 提供一个 PlaceholderAPI 变量
名称：
%tsl_blocks_placed_total%
返回：
玩家放置的所有方块的累计次数
若玩家无数据，返回 "0"
2️⃣ 放置方块总数的统计逻辑
使用原版的 Statistic.USE_ITEM 统计结构：
对每个 Material：
若 material.isBlock() 为 true
→ 则将 player.getStatistic(Statistic.USE_ITEM, material) 加入总和
伪代码：
total = 0
for each material in Material.values():
if material.isBlock():
total += player.getStatistic(USE_ITEM, material)
注意：不监听 BlockPlaceEvent
因为 USE_ITEM 对所有方块类物品 → 基本等同于“放置方块”的次数
且可以读到历史数据。
3️⃣ 性能要求（关键）
由于用于排行榜，查询频率不高（Topper 会设置刷新间隔），
因此插件实现可以是：
每次 Placeholder 被调用就重新计算一次
无需自定义缓存，也无需异步任务
不需要保存离线玩家数据
不需要自己的数据库
所有性能压力交给 Topper 的刷新频率控制（如 60s 刷新一次）。
